# Scriptwriter Skill (编剧技能)

此技能为 Scriptwriter 提供创作高质量、结构化剧本的专业知识，支持系列连续性和基于前作的续集创作。

## 技能组件

### 1. **screenplay-methodology.md** 📖

剧本创作方法论（参考用）：

- 三幕结构和故事弧线
- 角色发展原则
- 场景构建技巧
- 视觉化描述方法
- 系列连续性管理

### 2. **templates/episode-template.md**

- Episode 剧本的标准输出格式

### 3. **templates/series-continuity-guide.md**

- 系列连续性检查清单

## 用途

Scriptwriter 使用此技能来：

- 创作结构完整的剧本（包含足够的关键时刻用于 9 个 beats）
- 查看之前剧集的角色、情节、世界观
- 基于用户输入设想新一集的故事方向
- 保持系列角色、设定、风格的一致性
- 编写视觉化场景描述（便于分镜转换）
- 标注关键叙事时刻（beats 候选）

## 何时使用此技能

Scriptwriter 应在以下情况参考此技能：

- **创作新剧集前** - 查看之前剧集的 script/文件
- **规划故事** - 根据用户输入确定新一集的主题、冲突、发展
- **编写场景** - 参考 screenplay methodology 的视觉化描述方法
- **确保连续性** - 使用 series continuity guide 检查角色、设定一致性
- 不确定剧本格式时参考 episode template

## 系列连续性工作流

### 1. 查看之前的剧集

**查找现有剧本**:

```bash
# 查看script/目录下的所有剧集
ls script/
# 示例输出: ep01-awakening.md, ep02-revelation.md
```

**分析前作要点**:

- 主要角色及其发展
- 已建立的世界观设定
- 未解决的情节线
- 角色关系变化
- 视觉风格和氛围

### 2. 基于用户输入设想新剧集

**用户可能提供**:

- 主题关键词（如"背叛"、"发现真相"）
- 情节方向（如"角色 A 发现秘密"）
- 情感基调（如"悬疑"、"温馨"）
- 特定场景需求（如"在地铁站对峙"）

**Scriptwriter 的任务**:

1. **承接前作** - 确保角色状态、情节线与前作衔接
2. **发展故事** - 基于用户输入推进主线或支线
3. **设计转折** - 安排 9 个以上的关键叙事时刻
4. **视觉化描述** - 所有场景具体且可视化

### 3. 创作剧本

**必需元素**:

- 场景编号和位置（INT./EXT., 地点, 时间）
- 角色外观描述（首次出场时）
- 视觉化动作和场景细节
- 足够的关键时刻（≥9 个 beats 候选）

**输出文件命名**:

```
script/ep{XX}-{title}.md
例: script/ep03-betrayal.md
```

## 关键原则

### 1. 视觉优先

剧本必须便于分镜转换：

❌ **抽象描述**:

```
Emma感到困惑和恐惧。
```

✅ **视觉化描述**:

```
Emma的眼睛瞪大，手指颤抖着翻动笔记本的页面。汗珠从额头滑落。
她向后退了一步，背靠在墙上，呼吸急促。
```

### 2. 角色一致性

**建立角色档案** (从 ep01 开始):

- 规范外观描述
- 性格特质
- 说话方式
- 行为习惯

**在后续剧集中保持**:

- 外观描述逐字重复（除非故事需要改变）
- 性格发展合理渐进
- 语言风格一致

### 3. 结构清晰

**三幕结构** (适用每集):

**第一幕** (Setup, ~25%):

- 建立当前状态
- 介绍本集冲突
- 触发事件

**第二幕** (Confrontation, ~50%):

- 上升动作
- 障碍和复杂化
- 中点转折

**第三幕** (Resolution, ~25%):

- 高潮
- 决议
- 新常态（为下集埋伏笔）

### 4. Beats 分布

确保剧本包含**9 个以上**清晰的关键时刻：

**示例 beats 候选** (一个 30 页剧本):

- Beat 1 (p.3): 触发事件
- Beat 2 (p.7): 首次尝试
- Beat 3 (p.11): 遇到障碍
- Beat 4 (p.15): 发现线索
- Beat 5 (p.19): 中点转折
- Beat 6 (p.23): 低谷时刻
- Beat 7 (p.26): 最终对抗
- Beat 8 (p.28): 高潮
- Beat 9 (p.30): 新平衡

每个 beat 应该是视觉化的、情感充沛的转折点。

## 场景格式规范

### 标准场景头

```markdown
## 场景 {X} - {INT./EXT.} {地点} - {时间}

{场景建立描述 - 2-3 句，视觉化}

{角色动作和对话}
```

**示例**:

```markdown
## 场景 5 - INT. 地铁车厢内 - 夜晚

车厢内昏暗，白色荧光灯忽明忽暗。空荡的座椅上散落着几张废弃报纸。

EMMA（25 岁，银色长发，穿深红色大衣）坐在靠窗位置，翻开古旧的皮质笔记本。
里面密密麻麻的手写符号和发光的六芒星图案映照在她脸上。

突然，所有灯光熄灭。
```

### 角色首次出场

**必须包含**:

- 姓名（如有）
- 年龄范围
- 关键外观特征（头发、服装、标志物）

```markdown
MARCUS（50 多岁，灰白短发，海军蓝西装配红领带）推门进入。
```

### 动作描述

使用现在时，主动语态：

✅ **正确**:

```
Emma冲向门口。她的手抓住门把，用力推开。
```

❌ **错误**:

```
Emma应该向门口跑去，然后她会打开门。
```

## 系列连续性检查清单

创作新剧集前，检查：

- [ ] 已阅读所有之前的剧集
- [ ] 角色外观描述与前作一致
- [ ] 角色关系状态符合前作结尾
- [ ] 未解决情节线被考虑或推进
- [ ] 世界观设定无矛盾
- [ ] 视觉风格和氛围保持一致
- [ ] 新角色（如有）外观描述完整

## 输出约束

**严格禁止**:

- ❌ Frontmatter 元数据
- ❌ 模板说明
- ❌ 过于抽象的心理描写
- ❌ 缺少视觉细节的场景
- ❌ 角色外观描述矛盾

**必须包含**:

- ✅ 场景编号、位置、时间
- ✅ 首次出场角色的外观描述
- ✅ 视觉化场景和动作
- ✅ 足够的关键时刻（≥9 个 beats 候选）
- ✅ 三幕结构完整

## 技能激活

此技能对 Scriptwriter agent **始终可用**。创作剧本时必须参考此 skill 的原则和模板。
